# 9. Cloud Architecture

## 9-1. VM 생성 자동화 — Heat (오케스트레이션)

Heat를 이용해서 인프라를 **코드로 정의하고 자동 배포** 가능.

```
프레젠테이션 계층 (Web)
        +
애플리케이션 계층 (App)
        +
데이터 계층 (DB)
        ↓
Stack 단위로 한번에 자동 배포
```

---

## 9-2. HOT (Heat Orchestration Template)

- Heat의 배포 단위 템플릿
- **YAML 파일** 형식 사용
- **Main Template + Sub Template** 으로 구성

| 구성요소 | 역할 |
|---|---|
| **Main Template** | 전체 스펙 청사진 (어떤 자원을 배포할지 전체 정의) |
| **Sub Template** | 상세 구현 (각 자원의 세부 설정) |

**Sub Template 장점**

```
모듈 단위로 구성
→ 재사용 용이 ✅
→ 유지보수 편리 ✅
→ 가독성 향상 ✅
```

```yaml
# HOT 예시 구조
heat_template_version: 2018-08-31

description: 전체 스택 배포

resources:
  web_server:
    type: OS::Nova::Server    # Main Template
    properties:
      image: rhel8
      flavor: default

  db_server:
    type: my-db-template.yaml  # Sub Template (모듈)
```

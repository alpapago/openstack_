# Security

## SSH Keypair 기반 인증

---

### 사전 준비

```
① 클라이언트가 키페어 생성
   → 개인키(.pem) : 로컬에 보관 (절대 공유 금지)
   → 공개키       : OpenStack에 등록

② VM 생성 시 keypair 지정 (--key-name)
   → cloud-init이 VM 부팅 시 공개키를 자동으로 주입
   → 저장 위치: /home/cloud-user/.ssh/authorized_keys
```

---

### 접속 인증 흐름

```
# 사용자가 실행하는 명령어
ssh -i ~/.ssh/demo.pem cloud-user@<floating-ip>
         ↑
    개인키 지정 (SSH가 내부적으로 인증 처리)

① 클라이언트
   ssh -i demo.pem으로 접속 시도
        ↓
② Instance
   authorized_keys에서 공개키 목록 확인
   랜덤 Challenge 생성 → 클라이언트로 전송
        ↓
③ 클라이언트 (SSH가 자동 처리)
   demo.pem(개인키)으로 Challenge 서명 생성
   → 서명값만 전송 (개인키 자체는 전송 안함)
        ↓
④ Instance
   authorized_keys의 공개키로 서명값 검증
        ↓                    ↓
     맞음 ✅               틀림 ❌
        ↓                    ↓
   접속 허용            접속 거부
```

> `ssh -i demo.pem` = 개인키를 직접 전송하는 게 아니라
> SSH가 개인키로 서명을 생성해서 인증하는 것.
> 사용자는 개인키 파일만 지정하면 나머지는 SSH가 자동 처리.

---

### SSH 접속 명령어

```bash
# -i 옵션으로 개인키 지정
ssh -i <private-key> cloud-user@<floating-ip>
```

---

### 개인키 분실 시

- SSH 접속 **완전히 불가**
- Dashboard 콘솔 또는 VNC URL로만 접근 가능
- 비밀번호 기반 인증만 가능
- **해결 방법: 새 키페어로 VM 재배포**

> 이 실습 환경에서는 root 계정 비밀번호(redhat)로 접근 가능하지만
> 일반적인 운영 환경에서는 비밀번호 접근도 차단되어 있음

---

### 주의사항

- SSH 접속은 **Security Group에서 TCP 22번 포트 허용** 필요
- VM은 기본적으로 **cloud-user 계정 + 키 기반 인증**만 허용
- 개인키는 절대 네트워크로 전송하거나 공유하면 안됨
- 개인키 파일 권한 설정 필수

```bash
chmod 600 <private-key>  ← 본인만 읽기 가능하도록 설정
```

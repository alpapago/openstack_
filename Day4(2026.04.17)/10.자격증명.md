## 10-1. Application 인증

### Secret (1회성 인증 토큰)
- Application 인증을 위한 **1회성 인증 토큰**
- 일반 사용자 계정/패스워드 대신 사용
- 특정 권한만 부여 가능 → 보안상 유리

### clouds.yaml
- **여러 클라우드 환경을 하나의 파일로 관리**하는 설정 파일
- OpenStack CLI 사용 시 인증 정보를 파일로 관리

```yaml
# ~/.config/openstack/clouds.yaml
# OS_CLOUD=openstack 또는 --os-cloud=openstack 으로 사용

clouds:
  openstack:
    auth:
      auth_url: http://172.25.250.50:5000
      application_credential_id: "4392edbe4f8843c8af7b3cece6d89639"
      application_credential_secret: "Umx6R_vrDpqFW4WoXb7rsD-4672reni9qZkcfpQgFWXllavc6HDfg"
    region_name: "regionOne"
    interface: "public"
    identity_api_version: 3
    auth_type: "v3applicationcredential"
```

**사용 방법**

```bash
# 환경변수로 지정
export OS_CLOUD=openstack

# 또는 명령어 옵션으로 지정
openstack --os-cloud=openstack server list
```

**일반 rc 파일 vs clouds.yaml 비교**

| 구분 | rc 파일 | clouds.yaml |
|---|---|---|
| **관리 방식** | 환경변수 | 파일 기반 |
| **다중 클라우드** | 파일별로 관리 | 하나의 파일로 관리 |
| **인증 방식** | 계정/패스워드 | Application Credential 등 다양 |

## 10-2. OpenStack 업데이트 시 Down Time 발생 항목

아래 변경 사항은 **VM 재시작(Down Time)이 필요**합니다.

| 항목 | 설명 |
|---|---|
| **Flavor 변경** | vCPU, RAM, Disk 스펙 변경 시 재시작 필요 |
| **Image 변경** | OS 이미지 변경 시 재시작 필요 |
| **Subnet 변경** | IP 대역, DNS 등 네트워크 설정 변경 시 재시작 필요 |

> 운영 환경에서 위 항목 변경 시
> **사전에 Down Time 계획**을 수립하고 진행해야 함.
